# Nginx Cheat Sheet

Production-ready Nginx configurations for running as a reverse proxy in front of containerised or locally-running services.

You can format and validate your Nginx config at [nginxbeautifier.github.io](https://nginxbeautifier.github.io/).

> **Versions:** nginx 1.24+ · OpenSSL 3.x

## HTTPS Reverse Proxy with Podman endpoints 🏠 Personal homelab config

Nginx running on the host terminates TLS and forwards to services running in Podman containers on localhost. Each service gets its own `server` block - replace the certificate paths and domain names to match your setup.

```nginx
events
{
    worker_connections 4096;
}

http
{
    upstream gitea
    {
        server 127.0.0.1:3000 fail_timeout=0;
    }

    upstream jenkins
    {
        server 127.0.0.1:8080 fail_timeout=0;
    }

    upstream nexus
    {
        server 127.0.0.1:8081 fail_timeout=0;
    }

    server
    {
        listen 443 ssl;
        server_name gitea.libredevops.org;

        ssl_certificate     /etc/nginx/ssl/fullchain.cer;
        ssl_certificate_key /etc/nginx/ssl/wildcard.libredevops.org.key;

        location /
        {
            proxy_pass              http://gitea;
            proxy_http_version      1.1;
            proxy_set_header        Host                $host;
            proxy_set_header        X-Real-IP           $remote_addr;
            proxy_set_header        X-Forwarded-For     $proxy_add_x_forwarded_for;
            proxy_set_header        X-Forwarded-Proto   $scheme;
            proxy_set_header        Upgrade             $http_upgrade;
            proxy_set_header        Connection          "upgrade";
            proxy_redirect          http:// https://;
        }
    }

    server
    {
        listen 443 ssl;
        server_name jenkins.libredevops.org;

        ssl_certificate     /etc/nginx/ssl/fullchain.cer;
        ssl_certificate_key /etc/nginx/ssl/wildcard.libredevops.org.key;

        location /
        {
            proxy_pass              http://jenkins;
            proxy_http_version      1.1;
            proxy_set_header        Host                $host;
            proxy_set_header        X-Real-IP           $remote_addr;
            proxy_set_header        X-Forwarded-For     $proxy_add_x_forwarded_for;
            proxy_set_header        X-Forwarded-Proto   $scheme;
            proxy_set_header        Upgrade             $http_upgrade;
            proxy_set_header        Connection          "upgrade";
            proxy_redirect          http:// https://;

            # Required for the Jenkins CLI over SSL
            proxy_buffering         off;
            proxy_request_buffering off;

            # Workaround for JENKINS-45651
            add_header 'X-SSH-Endpoint' 'jenkins.libredevops.org:50022' always;
        }
    }

    server
    {
        listen 443 ssl;
        server_name nexus.libredevops.org;

        ssl_certificate     /etc/nginx/ssl/fullchain.cer;
        ssl_certificate_key /etc/nginx/ssl/wildcard.libredevops.org.key;

        location /
        {
            proxy_pass              http://nexus;
            proxy_http_version      1.1;
            proxy_set_header        Host                $host;
            proxy_set_header        X-Real-IP           $remote_addr;
            proxy_set_header        X-Forwarded-For     $proxy_add_x_forwarded_for;
            proxy_set_header        X-Forwarded-Proto   $scheme;
            proxy_set_header        Upgrade             $http_upgrade;
            proxy_set_header        Connection          "upgrade";
            proxy_redirect          http:// https://;
        }
    }
}
```

> **See also:** [Containers](/docs/cheatsheets/containers-cheatsheet) for the Podman/Docker endpoints these configs proxy. [Security](/docs/cheatsheets/security-cheatsheet) for openssl TLS certificate inspection and cipher testing.

## HTTP redirect to HTTPS ✅ Required - always redirect plaintext HTTP to HTTPS

Force all HTTP traffic to HTTPS with a 301 redirect. Add this server block alongside your SSL blocks.

```nginx
server
{
    listen 80;
    server_name _;
    return 301 https://$host$request_uri;
}
```

## Useful Nginx commands

```bash
nginx -t                    # test configuration syntax
nginx -s reload             # reload without downtime
nginx -s stop               # fast shutdown
systemctl status nginx      # service status
tail -f /var/log/nginx/access.log   # stream access log
tail -f /var/log/nginx/error.log    # stream error log
```

## Anti-patterns

- **No HTTP→HTTPS redirect** - without a port 80 block returning 301, browsers and crawlers will be served plaintext. Always add the redirect block.
- **`ssl_protocols` including `TLSv1` or `TLSv1.1`** - deprecated protocols with known weaknesses (BEAST, POODLE). Use only `ssl_protocols TLSv1.2 TLSv1.3;`.
- **No `proxy_read_timeout` / `proxy_send_timeout`** - without these, a hung upstream holds a worker connection open indefinitely and exhausts the connection pool under load. Set explicit timeouts.
- **No security response headers** - missing `X-Frame-Options`, `X-Content-Type-Options`, and `Content-Security-Policy` exposes the app to clickjacking and MIME-sniffing attacks without any browser-level mitigation.
- **Wildcard `server_name _` on an SSL vhost** - SSL requires SNI matching; a catchall HTTPS block will serve the wrong certificate for unrecognised hostnames. Use explicit `server_name` values for every TLS block.
- **Buffering enabled for streaming responses** - when proxying SSE, WebSocket upgrades, or chunked transfer responses, `proxy_buffering on` (the default) delays delivery. Set `proxy_buffering off` for those locations.

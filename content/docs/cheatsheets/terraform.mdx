# Terraform Cheat Sheet

A practical reference for common Terraform patterns used in Azure infrastructure. Each snippet solves a real problem — not just syntax demos.

## Formatting & String Manipulation

### Number padding

Use `format("%02d", count.index + 1)` to produce zero-padded numbers (`01`, `02`, ...) in resource names. This keeps names sortable and visually consistent when you have more than nine instances.

```hcl
resource "azurerm_application_security_group" "with_pad" {
  count               = 4
  name                = "asg-${var.short}-${var.loc}-${terraform.workspace}-web-${format("%02d", count.index + 1)}"
  location            = local.location
  resource_group_name = azurerm_resource_group.example_rg.name
  tags                = local.tags
}

resource "azurerm_application_security_group" "without_pad" {
  count               = 4
  name                = "asg-${var.short}-${var.loc}-${terraform.workspace}-web-${count.index + 1}"
  location            = local.location
  resource_group_name = azurerm_resource_group.example_rg.name
  tags                = local.tags
}
```

#### Output comparison

```text
Changes to Outputs:

      + "asg-lbdo-euw-tst-web-01",
      + "asg-lbdo-euw-tst-web-02",
      + "asg-lbdo-euw-tst-web-03",
      + "asg-lbdo-euw-tst-web-04",
    ]
  + asg_without_pad_output = [
      + "asg-lbdo-euw-tst-web-1",
      + "asg-lbdo-euw-tst-web-2",
      + "asg-lbdo-euw-tst-web-3",
      + "asg-lbdo-euw-tst-web-4",
    ]
```

### Title-case and strip punctuation

Combine `title()` and `replace()` to normalise a string — e.g. turning a kebab-case resource name into a PascalCase display name for tags or Azure AD application names.

```hcl
replace(replace(title("rg-craig-test"), "-", ""), " ", "")
# => "RgCraigTest"
```

### Region name lookup

Keep Azure region names consistent by storing shorthand ↔ longhand mappings in a variable and resolving them with `lookup()`. The third argument is a safe fallback if the key is not found.

```hcl
variable "loc" {
  description = "Shorthand Azure location — passed as TF_VAR in CI."
  type        = string
  default     = "ukw"
}

variable "regions" {
  type = map(string)
  default = {
    uks = "UK South"
    ukw = "UK West"
    eus = "East US"
  }
  description = "Converts shorthand name to longhand via map lookup."
}

locals {
  location = lookup(var.regions, var.loc, "UK South")
}
```

#### Output

```text
Changes to Outputs:
  + location_output = "UK West"
```

## Resource Patterns

### Conditional `for_each` with regex

Only create resources where a map value matches a pattern — everything else is skipped. This avoids maintaining a separate allow-list of which keys to deploy, and keeps the logic co-located with the data.

```hcl
variable "environment" {
  default     = "prd"
  type        = string
  description = "Used as an alternative to terraform.workspace."
}

locals {
  names = {
    key0 = var.environment          # "prd"
    key1 = "${var.environment}-vm"  # "prd-vm"
    key2 = "prd-biscuit"
    key3 = "tst_pizza"
  }
}

resource "azurerm_resource_group" "test_rg" {
  # Creates RGs only for values containing "prd-" — key1 and key2
  for_each = {
    for key, value in local.names : key => value
    if length(regexall("${var.environment}-", value)) > 0
  }
  location = local.location
  name     = each.value
}
```

#### Output — only matching keys are planned

```text
# azurerm_resource_group.test_rg["key1"] will be created
  + resource "azurerm_resource_group" "test_rg" {
      + id       = (known after apply)
      + location = "uksouth"
      + name     = "prd-vm"
    }

# azurerm_resource_group.test_rg["key2"] will be created
  + resource "azurerm_resource_group" "test_rg" {
      + id       = (known after apply)
      + location = "uksouth"
      + name     = "prd-biscuit"
    }
```

### Dynamic blocks — multiple identity type conditions

Terraform only allows one `identity` block per resource, but `SystemAssigned`, `UserAssigned`, and `SystemAssigned, UserAssigned` each need different attributes. Dynamic blocks let you express all three cases — only one will actually emit a block per run, depending on the variable values.

```hcl
dynamic "identity" {
  for_each = length(var.identity_ids) == 0 && var.identity_type == "SystemAssigned" ? [var.identity_type] : []
  content {
    type = var.identity_type
  }
}

dynamic "identity" {
  for_each = length(var.identity_ids) > 0 || var.identity_type == "UserAssigned" ? [var.identity_type] : []
  content {
    type         = var.identity_type
    identity_ids = length(var.identity_ids) > 0 ? var.identity_ids : []
  }
}

dynamic "identity" {
  for_each = length(var.identity_ids) > 0 || var.identity_type == "SystemAssigned, UserAssigned" ? [var.identity_type] : []
  content {
    type         = var.identity_type
    identity_ids = length(var.identity_ids) > 0 ? var.identity_ids : []
  }
}
```

### VMSS — Azure DevOps agent extension

The extension block for attaching a self-hosted Azure Pipelines agent to a Virtual Machine Scale Set. `jsonencode` handles the escaping so the settings block stays readable in HCL.

```hcl
extension {
  auto_upgrade_minor_version = false
  automatic_upgrade_enabled  = false
  name                       = "Microsoft.Azure.DevOps.Pipelines.Agent"
  provision_after_extensions = []
  publisher                  = "Microsoft.VisualStudio.Services"
  settings = jsonencode({
    agentDownloadUrl        = "https://vstsagentpackage.azureedge.net/agent/2.209.0/vsts-agent-linux-x64-2.209.0.tar.gz"
    agentFolder             = "/agent"
    enableScriptDownloadUrl = "https://vstsagenttools.blob.core.windows.net/tools/ElasticPools/Linux/13/enableagent.sh"
    isPipelinesAgent        = true
  })
  type                 = "TeamServicesAgentLinux"
  type_handler_version = "1.22"
}
```

## Type Conversions & Data Structures

### `for_each` output as `list(map(object({})))`

When resources are created with `for_each`, the result is a `map(object({}))`. This pattern shows how to query and reshape that output for downstream consumption.

```hcl
variable "environment" {
  default     = "prd"
  type        = string
  description = "Used as an alternative to terraform.workspace."
}

locals {
  names = {
    key0 = var.environment
    key1 = "${var.environment}-vm"
    key2 = "prd-biscuit"
    key3 = "tst_pizza"
  }
}

resource "azurerm_resource_group" "test_rg" {
  for_each = {
    for key, value in local.names : key => value
    if length(regexall("${var.environment}-", value)) > 0
  }
  location = local.location
  name     = each.value
}

output "rg_name" {
  value = element(azurerm_resource_group.test_rg[*], 0)
}
```

#### Output

```text
rg_name = [
  + {
      + key1 = {
          + id       = (known after apply)
          + location = "uksouth"
          + name     = "prd-vm"
          + tags     = null
          + timeouts = null
        }
      + key2 = {
          + id       = (known after apply)
          + location = "uksouth"
          + name     = "prd-biscuit"
          + tags     = null
          + timeouts = null
        }
    },
]
```

### Extract a single attribute from `map(object({}))`

Get just the `name` field from each entry in the map using a `for` expression.

```hcl
output "rg_name" {
  value = {
    for key, value in element(azurerm_resource_group.test_rg[*], 0) : key => value.name
  }
}
```

```text
rg_name = {
  + key1 = "prd-vm"
  + key2 = "prd-biscuit"
}
```

### Pass a map attribute into another resource

Extract `location` from the first map entry and pass it as input to a dependent resource. `element(values(...), 0)` picks the first value without needing to know the key name at authoring time.

```hcl
locals {
  resource_group_locations = {
    for key, value in element(azurerm_resource_group.test_rg[*], 0) : key => value.location
  }

  resource_group_name = {
    for key, value in element(azurerm_resource_group.test_rg[*], 0) : key => value.name
  }
}

resource "azurerm_application_security_group" "example" {
  name                = "libre-devops-asg"
  location            = element(values(local.resource_group_locations), 0)
  resource_group_name = element(values(local.resource_group_name), 0)

  tags = {
    Hello = "World"
  }
}

output "asg_location" {
  value = azurerm_application_security_group.example.location
}

output "asg_rg_name" {
  value = azurerm_application_security_group.example.resource_group_name
}
```

```text
+ asg_location = "uksouth"
+ asg_rg_name  = "prd-vm"
```

### Access nested identity inside a multi-key `for_each`

When each `for_each` value has its own nested blocks (like `identity`), use `element()` on the splat expression and then index into the nested block list.

```hcl
locals {
  fnc_apps = {
    fnc_app1 = { name = "fnc_app1" }
    fnc_app2 = { name = "fnc_app2" }
  }
}

resource "azurerm_function_app" "fnc" {
  for_each = local.fnc_apps

  identity {
    type = each.value.identity
  }
}

output "managed_identity_principal_id" {
  value = {
    for key, value in element(azurerm_function_app.fnc[*], 0) : key => element(value.identity, 0).principal_id
  }
}
```

```text
managed_identity_principal_id = {
  + fnc_app1 = "3ca56017-d384-4899-bbad-1066800809c0"
  + fnc_app2 = "0cca0226-011d-444d-8763-e210878ef4dc"
}
```

## Networking & Infrastructure Utilities

### Fetch your outbound IP

Useful for dynamically allowing a runner or developer machine IP in firewall rules, so you don't hard-code IPs or open wide CIDR ranges in CI.

`chomp()` strips the trailing newline from the HTTP response body — without it the IP includes a `\n` that breaks string comparisons and CIDR notation.

```hcl
data "http" "user_ip" {
  url = "https://ipv4.icanhazip.com"
}

data "http" "user_ip_from_aws" {
  url = "https://checkip.amazonaws.com"
}

output "my_ip" {
  value = data.http.user_ip.body
}

output "my_ip_chomp" {
  value = chomp(data.http.user_ip.body)
}
```

```text
+ my_ip       = <<-EOT
      20.108.154.139
  EOT
+ my_ip_chomp = "20.108.154.139"
```

## OS & Environment Detection

### Detect the runner OS

Terraform doesn't expose the host OS natively. The trick is a cross-platform `external` data source: on Linux, `printf` outputs JSON inline; on Windows, a `.cmd` wrapper does the same.

Add `printf.cmd` to the repo root for Windows runners:

```bat
@echo off
echo {"os": "Windows"}
```

Then call it via `data "external"`:

```hcl
data "external" "os" {
  working_dir = path.module
  program     = ["printf", "{\"os\": \"Linux\"}"]
}

locals {
  os    = data.external.os.result.os
  check = local.os == "Windows" ? "We are on Windows" : "We are on Linux"
}

output "os" {
  value = local.os
}
```

### Generate timestamp tags without `timestamp()`

`timestamp()` is evaluated at plan time and surfaces as "not known until apply" — this causes every plan/apply cycle to show a diff on tag values. Instead, use an `external` data source to call a small script that returns the current time as a JSON string.

```hcl
data "external" "detect_os" {
  working_dir = path.module
  program     = ["printf", "{\"os\": \"Linux\"}"]
}

data "external" "generate_timestamp" {
  program = data.external.detect_os.result.os == "Linux" ? [
    "${path.module}/timestamp.sh"
  ] : ["powershell", "${path.module}/timestamp.ps1"]
}

locals {
  dynamic_tags = {
    "LastUpdated" = data.external.generate_timestamp.result["timestamp"]
    "Environment" = terraform.workspace
  }

  tags = merge(var.static_tags, local.dynamic_tags)
}

variable "static_tags" {
  type        = map(string)
  description = "Static tags merged with dynamic ones."
  default = {
    "CostCentre" = "671888"
    "ManagedBy"  = "Terraform"
    "Contact"    = "help@libredevops.org"
  }
}
```

#### `timestamp.sh`

```bash
#!/usr/bin/env bash
DATE=$(date '+%d-%m-%Y:%H:%M')
echo "{\"timestamp\": \"$DATE\"}"
```

#### `timestamp.ps1`

```powershell
#!/usr/bin/env pwsh
$date       = Get-Date -Format "dd-MM-yyyy:HH:mm"
$jsonOutput = @{ timestamp = $date } | ConvertTo-Json
Write-Output $jsonOutput
```

#### `timestamp.py`

```python
import datetime

now       = datetime.datetime.now()
timestamp = now.strftime("%d-%m-%Y:%H:%M")
print("{\"timestamp\": \"" + timestamp + "\"}")
```

## State Management

### Force-unlock a stuck state

When a plan fails mid-run and leaves a lock, grab the lock ID from the error output and immediately release it. The `-force` flag skips the interactive confirmation prompt.

```powershell
$lockId = (terraform plan 2>&1 |
  Select-String -Pattern 'ID:\s+([\w-]+)' |
  ForEach-Object { $_.Matches.Groups[1].Value })
terraform force-unlock -force $lockId
```

```bash
lockId=$(terraform plan 2>&1 | grep -oP 'ID:\s+\K[\w-]+')
terraform force-unlock -force "$lockId"
```

### `override.tf` for local development

Swap the remote backend for a local run without touching the shared config. Terraform merges `override.tf` on top of the main config at runtime, so you can change the backend without modifying the file that CI uses. Add `override.tf` to `.gitignore`.

```hcl
terraform {
  required_providers {
    azurerm = {
      source = "hashicorp/azurerm"
    }
  }
  backend "azurerm" {
    subscription_id      = "your-subscription-id"
    storage_account_name = "your-storage-account"
    container_name       = "tfstate"
    key                  = "local.terraform.tfstate"
  }
}
```

## Workflow Automation

### Full local pipeline — init → validate → plan → compliance → tfsec → checkov

A single shell function that wires up the full local verification pipeline. Run it from the module root to validate, plan, check naming convention compliance, and scan for misconfigurations in one shot.

```bash
terraform_run() {
  rm -rf .terraform tfplan* terraform.lock.hcl

  if command -v tfenv &>/dev/null && \
     command -v terraform &>/dev/null && \
     command -v terraform-compliance &>/dev/null && \
     command -v tfsec &>/dev/null && \
     command -v checkov &>/dev/null; then
    echo "All packages are installed"
  else
    echo "Packages needed to run are not installed, exiting" && return 1
  fi

  terraform_workspace="prd"
  checkov_skipped_tests=""
  terraform_compliance_policy_path="git:https://github.com/libre-devops/azure-naming-convention.git//?ref=main"
  terraform_version="1.5.5"

  setup_tfenv() {
    if [ -z "${terraform_version}" ]; then
      export terraform_version="latest"
    fi
    tfenv install "${terraform_version}" && tfenv use "${terraform_version}"
  }

  terraform_plan() {
    terraform init && \
      terraform workspace new "${terraform_workspace}" || \
      terraform workspace select "${terraform_workspace}"
    terraform validate && \
      terraform fmt -recursive && \
      terraform plan -out "$(pwd)/tfplan.plan"
    terraform show -json tfplan.plan | tee tfplan.json >/dev/null
  }

  terraform_compliance_check() {
    terraform-compliance -p "$(pwd)/tfplan.json" -f "${terraform_compliance_policy_path}"
  }

  tfsec_check() {
    tfsec . --force-all-dirs
  }

  checkov_check() {
    checkov -f tfplan.json --skip-check "${checkov_skipped_tests}"
  }

  cleanup_tfplan() {
    rm -rf "$(pwd)/tfplan" "$(pwd)/tfplan.json"
  }

  setup_tfenv && \
    terraform_plan && \
    terraform_compliance_check && \
    tfsec_check && \
    checkov_check
  cleanup_tfplan
}
```
